# Service configuration management through UX

Allow users to add, edit, and delete service configurations for projects through the web UI instead of manually editing config.json

## Acceptance Criteria

- Users can view all services for a project
- Users can add new services with all config options
- Users can edit existing service configurations
- Users can delete services
- Changes persist to config.json
- Running services can be reconfigured (with restart prompt if needed)
